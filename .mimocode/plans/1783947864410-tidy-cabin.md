# Fix: Immich API Key Newline Crash

## Problem

The app crashes with `IllegalArgumentException: Unexpected char 0x0a at 40 in x-api-key value` when the user selects an album after entering their Immich API key. The API key stored in SharedPreferences contains a trailing newline character (`\n`), which OkHttp rejects in HTTP header values. This happens on Android TV because copy-paste / on-screen keyboard behavior appends a newline.

**Crash reproduced twice today** (13:59:52 and 14:03:43) in device dropbox logs.

## Fix: Trim the API key at consumption points

### 1. `ImmichRepository.kt` â€” add a private trimmed accessor

**File:** `app/src/main/java/com/neilturner/aerialviews/providers/immich/ImmichRepository.kt`

Add a private property near the top of the class:
```kotlin
private val apiKey: String get() = prefs.apiKey.trim()
```

Then replace all `prefs.apiKey` references with `apiKey` (used in ~8 API calls: `getAlbums`, `getAlbumsV3`, `getAlbum`, `getAlbumAssets`, `getFavoriteAssets`, `getRandomAssets`, `getRecentAssets`).

### 2. `ImagePlayerHelper.kt` â€” trim in the OkHttp interceptor

**File:** `app/src/main/java/com/neilturner/aerialviews/ui/core/ImagePlayerHelper.kt`

At line 85, change:
```kotlin
.addHeader("X-API-Key", ImmichMediaPrefs.apiKey)
```
to:
```kotlin
.addHeader("X-API-Key", ImmichMediaPrefs.apiKey.trim())
```

### 3. `.gitignore` â€” add `.mimocode/`

**File:** `.gitignore`

Add `.mimocode/` under the existing `# Agents` section (after `.qwen/`).

## Tests

**Existing tests need no changes** â€” `ImmichRepositoryTest.kt` mocks `prefs.apiKey` to return clean strings (`""`, `"test-api-key"`), so `.trim()` is a no-op and all 15+ tests continue to pass.

**Add one new test** to `ImmichRepositoryTest.kt` in the `FetchAlbums` nested class to verify trimming:

```kotlin
@Test
fun `trims whitespace from API key before calling API`() = runTest {
    every { prefs.apiKey } returns "test-api-key\n"
    coEvery { api.getServerVersion() } returns Response.success(serverVersionResponse(2))

    val albums = listOf(Album(id = "a1", name = "Album 1"))
    coEvery { api.getAlbums(apiKey = "test-api-key") } returns Response.success(albums)
    coEvery { api.getAlbums(apiKey = "test-api-key", shared = true) } returns Response.success(emptyList())

    val result = repository.fetchAlbums()

    assertTrue(result.isSuccess)
    coVerify { api.getAlbums(apiKey = "test-api-key") }
}
```

This confirms that a trailing newline in the prefs value is stripped before the API call.
